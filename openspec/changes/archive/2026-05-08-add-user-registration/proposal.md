## Why

当前后端用户模块只有查询能力，没有新增或修改用户的入口，前端也缺少注册页面，无法支持基础注册流程。现在需要补齐用户注册能力，并在注册成功后通过 RabbitMQ 触发短信发送链路，以便为后续真实消息通知接入预留系统边界。

## What Changes

- 在后端新增用户注册接口，要求提交用户名和手机号完成注册。
- 在后端新增用户写入能力，保存新注册用户，并为用户补齐当前领域数据结构需要的字段。
- 在后端引入 RabbitMQ 消息发布边界，在注册成功后发送“注册成功短信”消息。
- 提供短信发送占位实现，使用打印方法替代真实短信下发。
- 在前端新增用户注册页面，允许输入用户名和手机号并提交注册。
- 调整前后端集成方式，使前端可调用后端注册接口并展示注册结果。

## Capabilities

### New Capabilities
- `user-registration`: 用户注册流程、注册数据校验、消息发送触发与前端注册页面
- `message-notification`: RabbitMQ 消息发布与短信占位发送能力

### Modified Capabilities
- `user-management`: 用户模块从只读查询扩展为支持注册写入能力
- `graphql-api`: GraphQL 接口从只读查询扩展为支持用户注册变更

## Impact

- 受影响代码包括 `backend/src/user/`、Prisma 数据模型、GraphQL schema、前端页面与 API 调用逻辑。
- 后端新增 RabbitMQ 相关依赖或封装，并增加注册成功后的消息发布流程。
- 前端新增注册表单页面与提交交互。
