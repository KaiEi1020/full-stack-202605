## ADDED Requirements

### Requirement: 系统必须支持岗位需求配置
前端必须提供岗位需求编辑区域，使用户能够输入岗位描述、必备技能和加分技能，并将其作为评分分析输入。

#### Scenario: 用户创建一个岗位需求
- **WHEN** 用户填写岗位描述、必备技能和加分技能并提交
- **THEN** 系统必须将该岗位需求持久化保存到 SQLite 数据库并允许后续对候选人执行匹配评分

### Requirement: 系统必须输出标准化匹配评分结果
系统必须针对候选人与岗位需求的匹配结果返回 0 到 100 的综合评分，以及技能匹配度、经验相关性、教育背景契合度三个维度的子评分，并附带一段说明候选人优势与不足的 AI 评语。

#### Scenario: 某候选人完成岗位匹配分析
- **WHEN** 用户对某位候选人和某个岗位需求发起评分
- **THEN** 系统必须返回综合评分、三个子评分和 AI 评语

#### Scenario: 评分缺少某个维度依据
- **WHEN** 候选人简历缺少教育或经历信息导致分析依据不足
- **THEN** 系统必须仍然返回完整评分结构并在评语中体现不确定性或短板说明

### Requirement: 系统必须通过 BigModel HTTP API 生成匹配评分
后端必须通过环境变量读取 BigModel API Key，并调用智谱 BigModel HTTP API 生成岗位匹配评分和评语，不得在前端暴露密钥。

#### Scenario: 服务端发起 BigModel 评分请求
- **WHEN** 用户对某位候选人和某个岗位需求发起评分
- **THEN** 系统必须使用服务端环境变量中的 API Key 调用 BigModel HTTP 接口生成评分结果

#### Scenario: 缺少 BigModel 配置导致无法评分
- **WHEN** 后端未读取到 BigModel API Key 且用户发起评分
- **THEN** 系统必须返回明确配置错误而不是伪造评分结果

### Requirement: 系统必须可视化展示评分结果
前端必须以图表形式展示综合评分与多维子评分，至少包含一种综合进度表现和一种多维对比表现。

#### Scenario: 查看候选人评分图表
- **WHEN** 用户打开已完成评分的候选人详情
- **THEN** 系统必须展示可视化图表以呈现综合分和维度分

### Requirement: 系统应支持多个岗位需求的并行评分对比
系统应允许用户使用多个岗位需求对同一候选人进行评分，并并排查看不同岗位下的结果。

#### Scenario: 候选人对多个 JD 评分
- **WHEN** 用户选择两个或更多岗位需求对同一候选人执行分析
- **THEN** 系统必须保留每个岗位需求对应的独立评分结果并支持切换或并排对比
