## ADDED Requirements

### Requirement: 系统必须提供组件化招聘工作台界面
前端必须以组件化方式组织招聘工作台，至少包括上传区、JD 编辑区、候选人列表区、候选人详情区和全局反馈区，并适配桌面端宽度不小于 1280px 的布局展示。

#### Scenario: 在桌面端打开招聘工作台
- **WHEN** 用户在桌面端访问该页面
- **THEN** 系统必须以清晰分区的响应式布局展示各功能面板

### Requirement: 系统必须围绕单文件上传流程组织界面
前端上传区域必须明确体现单个 PDF 文件选择、上传、解析与结果反馈流程，不得展示多文件队列交互。

#### Scenario: 页面显示上传入口
- **WHEN** 用户进入招聘工作台并查看上传区域
- **THEN** 系统必须展示单文件上传入口与当前文件处理状态而不是多文件列表

### Requirement: 系统必须提供统一 Loading 与错误反馈
系统必须为上传、解析、AI 提取、评分和候选人加载等异步流程提供可见的 Loading 态，并在失败时提供统一错误反馈。

#### Scenario: 候选人列表正在加载
- **WHEN** 前端尚未完成候选人数据加载
- **THEN** 系统必须展示加载态而不是空白页面

#### Scenario: SSE 提取请求失败
- **WHEN** 前端在订阅或处理提取事件流时收到失败结果
- **THEN** 系统必须展示统一错误反馈并允许用户重新触发流程

### Requirement: 系统应支持主题与细节体验增强
系统应支持亮色与暗色主题切换，并通过骨架屏、动画过渡或键盘快捷键提升交互体验。

#### Scenario: 用户切换深浅色主题
- **WHEN** 用户在页面中切换亮色与暗色主题
- **THEN** 系统必须更新全局视觉风格且保持核心内容可读

#### Scenario: 详情面板打开时展示过渡反馈
- **WHEN** 用户从列表打开候选人详情
- **THEN** 系统应通过骨架屏或动画过渡减少界面跳变感
