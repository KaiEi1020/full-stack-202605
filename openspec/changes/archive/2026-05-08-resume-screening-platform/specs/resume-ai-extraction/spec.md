## ADDED Requirements

### Requirement: 系统必须提取简历中的核心结构化信息
系统必须基于简历清洗文本提取以下结构化信息：姓名、电话、邮箱、所在城市、教育背景、工作经历、技能标签。系统可以扩展提取项目经历，但不得影响必选字段的输出结构。

#### Scenario: AI 提取成功返回完整结果
- **WHEN** 后端收到一份可解析的简历文本并完成 AI 提取
- **THEN** 系统必须返回包含基本信息、教育背景、工作经历和技能标签的结构化结果

#### Scenario: AI 未识别到部分字段
- **WHEN** AI 无法识别某些必选字段
- **THEN** 系统必须保留字段键并以空值或空数组表达缺失信息

### Requirement: 系统必须通过 SSE 流式返回提取过程
后端必须提供基于 SSE 的提取事件流，前端必须能够在提取过程中逐步接收阶段状态、局部字段或分段结果，并在完成前持续更新页面内容。

#### Scenario: 前端订阅简历提取事件流
- **WHEN** 用户对某位候选人发起 AI 信息提取
- **THEN** 系统必须建立 SSE 连接并持续发送该候选人的提取进度事件直到完成或失败

#### Scenario: 提取过程发生错误
- **WHEN** AI 提取过程中发生模型调用失败或解析错误
- **THEN** 系统必须通过 SSE 发送失败事件并让前端展示错误状态

### Requirement: 系统必须允许用户修正 AI 提取结果
前端必须允许用户手动编辑 AI 提取出的关键字段，并将修正后的结果保存为当前候选人的展示版本。

#### Scenario: 用户修改候选人邮箱或技能标签
- **WHEN** 用户在前端编辑结构化字段并提交保存
- **THEN** 系统必须保存用户修正后的字段并在后续详情展示中优先显示修正结果

### Requirement: 系统必须通过 BigModel HTTP API 完成提取
后端必须通过环境变量读取 BigModel API Key，并调用智谱 BigModel HTTP API 完成简历结构化提取，不得在前端暴露密钥。

#### Scenario: 服务端发起 BigModel 提取请求
- **WHEN** 后端收到可用于提取的简历清洗文本
- **THEN** 系统必须使用服务端环境变量中的 API Key 调用 BigModel HTTP 接口完成提取

#### Scenario: 缺少 BigModel 配置
- **WHEN** 后端启动后未读取到 BigModel API Key
- **THEN** 系统必须在调用提取能力时返回明确配置错误而不是伪造成功结果
