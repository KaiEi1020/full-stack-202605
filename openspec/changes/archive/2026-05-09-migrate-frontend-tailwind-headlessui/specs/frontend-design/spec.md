## ADDED Requirements

### Requirement: 前端必须重新优化页面视觉层次
系统 MUST 基于新的 UI 基础体系重新组织页面的视觉层次、间距、色彩和信息分区，使核心页面具备统一的设计语言。

#### Scenario: 打开核心页面
- **WHEN** 用户访问首页、上传页、候选人列表页、详情页、GraphQL 示例页或注册页
- **THEN** 页面必须展示统一的导航、卡片、内容区和操作区设计风格

### Requirement: 前端必须提供一致的状态与操作反馈样式
系统 MUST 为加载中、错误、空状态、主要操作和次要操作提供一致的视觉表达。

#### Scenario: 页面展示请求状态
- **WHEN** 任一页面进入加载、失败或空数据状态
- **THEN** 页面必须使用统一设计样式展示状态反馈信息
