## MODIFIED Requirements

### Requirement: 系统必须提供 RESTful 简历接口
系统 MUST 通过标准 HTTP RESTful 接口提供简历资源的创建、查询、更新与删除能力，并在简历列表与详情响应中返回前端展示管理状态所需的字段，包括解析状态以及在失败时可读的失败原因。

#### Scenario: 创建简历
- **WHEN** 客户端向简历集合资源发送合法的创建请求
- **THEN** 系统必须创建一条新的简历记录并返回创建结果

#### Scenario: 查询简历详情
- **WHEN** 客户端使用简历标识访问单个简历资源
- **THEN** 系统必须返回对应简历的完整字段数据或明确的不存在结果

#### Scenario: 更新简历
- **WHEN** 客户端向单个简历资源发送合法的更新请求
- **THEN** 系统必须持久化变更并返回更新后的简历结果

#### Scenario: 删除简历
- **WHEN** 客户端请求删除已有简历资源
- **THEN** 系统必须删除对应记录或按约定返回删除成功结果

#### Scenario: 查询简历列表中的解析失败记录
- **WHEN** 客户端请求简历列表且某些简历解析失败
- **THEN** 系统必须在对应记录中返回失败状态和可供前端直接展示的失败原因
