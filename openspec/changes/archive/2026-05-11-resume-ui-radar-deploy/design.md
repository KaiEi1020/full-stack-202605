## Context

当前简历工作台已经完成从 GraphQL/Prisma 到 REST/TypeORM 的迁移，核心功能可用，但前端导航与详情页交互仍存在体验割裂：评分图表有独立入口却又与详情页内容重叠，用户注册入口不属于当前主流程，详情页状态流转展示缺乏明确的当前状态反馈，而现有评分可视化仅为条形图，不能直观表达多维度结构。另外，部署相关脚本虽然已存在，但缺少与镜像打包联动的验证步骤，因此需要把 UI 收敛和远程启动验证一起纳入本次小范围收尾变更。

## Goals / Non-Goals

**Goals:**
- 删除评分图表导航 TAB，将评分展示收敛到简历详情页
- 隐藏用户注册 TAB，保留工作台主导航聚焦在简历流程
- 修复简历详情页状态流转展示问题，使当前状态与可操作状态更易理解
- 在简历详情页补充雷达图展示多维度评分
- 验证镜像打包与 `bash ./scripts/start-remote.sh` 执行链路

**Non-Goals:**
- 不重新设计整套导航系统
- 不改动后端评分算法与评分字段结构
- 不引入新的部署平台或编排方案
- 不扩展用户注册流程本身的业务能力

## Decisions

1. **导航收敛到主流程**
   - 决策：移除评分图表 TAB，隐藏用户注册 TAB，但保留对应页面实现，以降低回滚成本。
   - 原因：当前两个入口都不属于简历筛选主流程，且评分入口与详情页内容重复。
   - 备选方案：彻底删除页面；不采用，因为本次重点是 UI 收敛而不是功能清除。

2. **评分详情集中在简历详情页**
   - 决策：在简历详情页直接展示条形图 + 雷达图 + AI 评语，而不是依赖独立评分页。
   - 原因：用户查看单份简历时最需要同时看到状态、文本、PDF 与评分信息。
   - 备选方案：保留独立评分页并从详情页跳转；不采用，因为增加一步跳转且与本次删除 TAB 目标矛盾。

3. **状态流转采用当前状态高亮 + 操作按钮分离**
   - 决策：详情页展示当前状态标识，并将状态切换按钮设计为明确的操作区，避免把所有状态按钮都表现为同级当前状态。
   - 原因：现有展示容易让用户误判当前状态与可切换状态。
   - 备选方案：仅修改颜色；不采用，因为信息层级仍不清晰。

4. **雷达图使用轻量 SVG 自绘**
   - 决策：在前端使用现有 React 组件直接渲染 SVG 雷达图，不引入新的图表依赖。
   - 原因：当前只需要 4 维评分图，SVG 足够，且可控、轻量。
   - 备选方案：引入图表库；不采用，因为依赖成本高于需求复杂度。

5. **部署验证以现有脚本为准**
   - 决策：镜像打包后以 `bash ./scripts/start-remote.sh` 为验收命令，必要时修正脚本依赖的路径、镜像名或环境变量。
   - 原因：用户明确要求验证该命令而不是只完成本地构建。
   - 备选方案：仅执行镜像 build；不采用，因为不能覆盖远程启动链路。

## Risks / Trade-offs

- [隐藏入口后可发现性下降] 用户注册与独立评分页仍可访问但不在导航中暴露 → 保留页面路由，必要时可通过直链访问
- [状态展示重构引入误操作] 按钮层级调整可能改变现有交互习惯 → 用当前状态高亮和禁用/次级样式明确区分
- [SVG 雷达图实现偏差] 手写图形可能在边界值下显示不均匀 → 使用归一化坐标与固定维度顺序，并在 0/100 分场景下测试
- [远程启动依赖外部环境] `start-remote.sh` 可能依赖本地不存在的环境 → 先识别脚本前置条件并在本地验证可控部分

## Migration Plan

1. 调整 `App.tsx` 导航配置，移除评分图表与用户注册入口
2. 更新简历详情页状态流转区布局与当前状态展示
3. 扩展评分组件或新增雷达图组件，嵌入详情页
4. 本地构建前端并在浏览器验证详情页展示
5. 打包镜像并执行 `bash ./scripts/start-remote.sh` 验证脚本流程

## Open Questions

- `start-remote.sh` 是否依赖远程主机登录或外部环境变量
- 独立评分页在去掉 TAB 后是否还需要保留直链访问入口
