## MODIFIED Requirements

### Requirement: 后端模块依赖方向必须保持单向分层
系统 MUST 保证复杂模块中的依赖方向清晰，并统一通过 TypeORM 访问持久化数据；简单模块 MUST 保持控制器到服务到持久化访问的单向依赖。业务状态枚举等领域语义类型 MUST 定义在业务模块目录或共享业务类型目录中，而不是定义在 `core/database` 等基础设施目录中。

#### Scenario: 执行复杂模块请求
- **WHEN** REST 控制器处理复杂模块请求
- **THEN** 控制器必须调用应用层服务，并由基础设施层提供具体的数据访问实现

#### Scenario: 执行业务状态判断
- **WHEN** 业务模块需要声明或判断候选人、岗位、解析等状态类型
- **THEN** 系统必须从业务模块目录或共享业务类型目录引用对应枚举，而不是从 `core/database` 定义这些业务状态
